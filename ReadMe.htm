Instructions for using TraceWin

What is TraceWin?

TraceWin is a Windows applet that displays C++ (TRACE, ATLTRACE, _RPTX) and .NET (Trace.Write) diagnostics in a separate window. You can use it to help debug your app without running it under the debugger. You can also save the output to a file for later use.

To use TraceWin in an MFC app:

  • Copy the executable TraceWin.exe to a folder in your PATH.

  • Copy TraceWin.h to a folder in your project or INCLUDE path.

  • #include TraceWin.h in your main program file (eg. MyApp.cpp).
    IMPORTANT: Only one module per project should include TraceWin.h!

  • Compile your app with _DEBUG.

To use TraceWin in a .NET app:

  • Copy TraceWinListener.dll somewhere handy.

  • Add a reference to TraceWinListener.dll in your .NET project.

  • Add the following lines in your main module file (eg. Form1.cs): 

    using TraceWin;
Debug.Listeners.Add(new TraceWinListener());

    Use equivalent lines for Visual Basic and other languages.

  • Don't forget to compile your app with /d:TRACE !

To see trace output:

  • Run TraceWin.exe before starting your app.

To build TraceWin.exe and TraceWinListener.dll:

If all you want to do is use TraceWin, use the .exe and .dll provided. But if you want to modify either of these for whatever reason, or just see how they work, the source code is in src/cpp and src/cs. You can open the solution files in Visual Studio .NET, or just open a command prompt, cd to the appropriate folder and type: 

nmake
Make sure your PATH is set up to find Visual Studio .NET and NMAKE.
